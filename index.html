Five agents

Real boundaries,
not prose pleas.

In v2, scope was a paragraph asking an agent to please not build. In v3, if the Researcher lacks the Write tool, it cannot write, regardless of what any prompt says. Tool allowlists are the enforcement layer; prose is not.

Yellow · Researcher
claude-haiku-4-5 · fan-out

Scout mode: discovers opportunities and synthesizes research. Evaluate mode: grades live products with a KILL / PIVOT / SCALE verdict and supporting evidence.

Tools: Read, WebSearch, WebFetch, Bash (read-only: grep/find/curl). MCP: read-only, no write connectors.

Red-Orange · Designer
claude-sonnet-4-6

UX flows, system architecture, wireframes, and design specs. Translates research briefs into actionable blueprints the Maker can execute without ambiguity.

Tools: Read, Write, Edit. No MCP. Cannot deploy or publish.

Blue · Maker
claude-sonnet-4-6 · worktree

Code, tests, deployment. Works from specs; never redesigns without approval. Runs in git worktree isolation so in-progress builds cannot corrupt the main branch.

Tools: Read, Write, Edit, Bash, Glob, Grep. MCP: deploy-capable (Cloudflare, Stripe). Only role with this access.

Green · Marketer
claude-haiku-4-5 · variants

Copywriting, campaigns, growth assets, email sequences. Drafts distribution at high volume and low cost. A human sends; the Marketer never publishes autonomously.

Tools: Read, Write. No MCP. No publish action. STOP-gated always.

Purple · Manager
claude-opus-4-8 · apex

Decomposes tasks, dispatches agents, enforces quality gates, synthesizes outputs. Operates at the orchestration apex only. If Purple is writing code, something is wrong.

Tools: All (orchestration use only). MCP: read-only plus approvals. Delegates execution; never does specialist work itself.

Model routing cuts cost by ~75-80% vs running every stage at Opus. Haiku at $1/$5 per MTok for high-volume fan-out (research sweeps, copy variants). Sonnet at $3/$15 for build and design. Opus at $5/$25 for orchestration apex and final synthesis only. One field per stage: model:

Orchestration as code

The sprint pipeline
in 40 lines.

In v2, the sprint mode was prose asking one Opus instance to roleplay coordination. In v3, it is an executed Workflow script with barrier semantics, schema validation, and deterministic sequencing. Intelligence stays in the leaf agents; orchestration moves out of the token stream.

v3/workflows/sprint.workflow.js research-preview API 
// Workflow API: research-preview. Signatures provisional pending GA.
import { pipeline, agent, verify, gate } from '@octopus/core';
import { ResearchBrief, DesignSpec, BuildArtifact } from '@octopus/schemas';

export default pipeline({
  name: 'sprint',

  stages: [

    agent({
      type:   'researcher',        // Haiku fan-out: fast, cheap, wide
      model:  'claude-haiku-4-5',
      output: ResearchBrief,     // schema-validated handoff
    }),

    agent({
      type:   'designer',
      model:  'claude-sonnet-4-6',
      input:  ResearchBrief,
      output: DesignSpec,
    }),

    agent({
      type:      'maker',
      model:     'claude-sonnet-4-6',
      isolation: 'worktree',        // git-isolated working tree
      input:     DesignSpec,
      output:    BuildArtifact,
    }),

    verify({
      cold:     true,               // fresh context: cannot see builder's reasoning
      criteria: 'CRITERIA.md',     // frozen binary tests, written before the run
      on_fail:  'halt',            // pipeline stops; blocker list returned to builder
    }),

    gate({
      blocks: ['deploy', 'publish', 'send', 'delete'],
      writes: 'PENDING-APPROVALS.md',   // parked; waits for human
    }),

    agent({
      type:  'marketer',            // Haiku: high-volume copy variants
      model: 'claude-haiku-4-5',
      input: BuildArtifact,
    }),

  ],
});

Each stage receives a schema-validated artifact from the previous stage. The verify stage spawns a cold agent with independent context every round. The gate primitive halts and writes to PENDING-APPROVALS.md before any irreversible action.

The orchestration adapter compiles this graph to native Workflow primitives when available, and degrades to the v2 prose-Manager path on platforms that do not yet have the Workflow tool (Codex CLI, Gemini CLI, pre-GA Claude Code). Zero config change required on upgrade.

Two load-bearing primitives

It grades itself.

The Workflow tool gives you pipeline() and parallel(). Octopus gives you verify and gate. Neither requires anything beyond the plugin. These are the differentiating layer: the Workflow tool gives you determinism; Octopus gives you trustworthiness.

Cold Verify

Independent context, every round.

Anthropic's own June 2026 loop experiments found a separate verifier beats self-critique. A builder's context is compromised by intent; Octopus does not ask the builder to grade its own work.

  • Criteria are frozen at pipeline start with octopus.criteria([...]). Each criterion is binary: pass or fail, no partial credit.
  • A fresh cold agent is spawned per round. It sees only the frozen criteria and the artifact directory; never the builder's session, reasoning, or prior conversation.
  • Its system instruction: "Your job is to find failures. Approve only if every criterion passes without interpretation."
  • On failure, the pipeline halts and returns a blocker list. The next verify round is always a new cold agent, regardless of accumulated prior rounds.
  • Two non-disableable security checks run first: a secret scan of all staged files, and npm audit --audit-level=high (or equivalent). Any hit is an automatic fail.
STOP Gate

Human in the loop, exactly once.

Before any action that is irreversible, outward-facing, or involves real spend, the pipeline pauses. The decision stays parked until a human clears it. This cannot be bypassed by any agent instruction.

  • Triggers on: deploy, publish, send, delete, and any purchase action.
  • The pending action is written verbatim to PENDING-APPROVALS.md with full context for the reviewer.
  • The Marketer role is permanently STOP-gated. It drafts distribution assets; it never sends, posts, or publishes autonomously.
  • Gate is composable: add custom trigger patterns to your CRITERIA.md to block domain-specific irreversible actions.

Safe by design

Zero config collision.

Octopus's rarest property since v1: it ships no hooks, no settings.json, no CLAUDE.md mutations. v3 preserves this guarantee explicitly while moving to a plugin architecture with materially larger capability.

No global installation

Installs under ~/.claude/plugins/octopus/. Never touches your CLAUDE.md, settings.json, or any other config path. Uninstall with one command; no residual side effects.

Least privilege per role

Researcher has read-only MCP; no write connectors. Maker is the only role with deploy-capable MCP (Cloudflare, Stripe). The Verifier gets nothing but Read. Privilege is not a prose plea; it is the tool allowlist.

No autonomous distribution

The Marketer role drafts; a human sends. This is not a setting. It is a permanent STOP-gate on every outward-facing action from the Marketer. Auto-posting to social media or Product Hunt is blocked at the architecture level.

Security pass in every verify gate

Before any artifact can ship: a secret scan checks all staged files for tokens, API keys, and passwords. A dependency audit runs at high severity. Both checks are non-disableable by user criteria and cannot be overridden by any agent prompt.

$ cp octopus/octopus.md ./CLAUDE.md

# Start Claude Code in your project
$ claude

# Then at the prompt:
> sprint build a micro-tool that...

$ claude plugin remove octopus
